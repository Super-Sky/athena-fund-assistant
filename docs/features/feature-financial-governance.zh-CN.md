# 金融治理门

## 范围

基金分析接口在返回决策矩阵前，会用确定性的产品级治理门进行评估。该治理门保留具体决策支持的价值，同时保持由用户而不是系统选择并执行动作的边界。

## 规则

- 阻断收益承诺、自动下单执行，以及“必须买入”“必须卖出”这类绝对指令。
- 阻断少于两个方案的输出。
- 标记任一方案缺少风险、反证条件或复盘时间的输出。
- 阻断没有画像、组合、模板、规则或回测/模拟依据的非零仓位调整比例。
- 标记缺少或格式不正确的来源、provider、抓取时间、市场时间或时区 metadata。
- 返回机器可读的 `passed`、`flagged`、`blocked` 检查结果。`blocked` 输出不会由 `POST /api/analysis/fund` 交付；`flagged` 输出仍可交付，但必须保留披露信息。

## 边界

- 这是确定性输出护栏，不是投资适当性审查，也不构成持牌投顾服务。
- 不执行交易、不保存券商凭证，也不接收券商下单指令。
- 不对数据质量、授权或新鲜度作认证；它要求这些属性必须显式存在，供 UI 和用户判断。

## 验证

- `go test ./internal/governance ./internal/decision ./internal/server`
- 治理测试覆盖允许输出、缺失 metadata 与风险/反证条件的标记、禁止措辞，以及缺失比例依据时的阻断。
